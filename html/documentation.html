PLAYER

  • -id -> 0 reserved for "neutral", standard players are 1-16
  • -team -> reference to team object the player is a part of
  • -name -> duh
  • -score -> score of the individual player, team score is separate and total of all players
  • -isBot -> boolean of whether player is a bot or not
  • -getOwned(map) -> pass copy of map and get a list of planets owned by this player
  • -getTeamOwned(map) -> pass copy of map and get list of planets owned by this players team
  • -getTeamId()-> returns integer id of this players team

TEAM

  • -id-> same as Player class, numeric from 1-16
  • -players-> get list of players on this team
  • -getScore()->gets total score of this team

BOT

Extends player class, your bot will extend bot class
  • -makeMove(gameMap, teams) -> The function you decide what move your bot makes. Receives a reference to the game map as a parameter
  • -onGameStart(gameMap, teams) -> The function called when the map is first loaded in, for any initial strategy decisions. Receives reference to game map.
  • -mapUpdate(gameMap, teams, action) -> The function called after each turn is made.
You may add any additional functions / variables, just ensuring that at least above functions are implemented.

PLANET

  • -id-> numeric id of the planet, from 0+
  • -owner-> reference to player that owns this planet
  • -value-> integer point value of the planet, 1 by default
  • -defense-> integer value of how many "ships" / defense is on the planet (prod mode)
  • -connections-> list of connected planet ids
  • -shieldVal-> value of shield on planet, any value greater than 0 means shield
  • -bonerShowing-> poorly named boolean for whether the planet is part of an activated bonus
  • -ownerId-> id of player who owns planet
  • -teamId-> id of team of player who owns planet. Reference this over ownerId to maximise compatibility with teams modes
  • -hasShield-> returns boolean of whether or not planet has shield up
  • -isLocked-> returns boolean of whether planet is locked
